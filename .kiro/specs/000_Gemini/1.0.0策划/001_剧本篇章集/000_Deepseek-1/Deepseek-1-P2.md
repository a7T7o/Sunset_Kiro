# 篇章二：核心玩法循环详细设计

## 一、日常循环设计

### 1.1 一天的时间结构

游戏内一天对应现实时间约 **20~25分钟**（取决于玩家操作效率），采用 `TimeManager` 的每10分钟触发一次事件机制。关键时间节点如下：

| 游戏时间 | 事件/机制 | 备注 |
|----------|----------|------|
| 6:00 | 玩家自动起床（如果前一天睡在合法床上） | 触发 `SleepEvent` 结束，新一天开始 |
| 6:00 - 20:00 | 白天时段，光线充足 | 光照强度随时间变化 |
| 8:00 | 商店开门（如果NPC无特殊事件） | 可交易 |
| 12:00 - 13:00 | NPC午餐时间（部分NPC回屋休息） | 社交机会减少 |
| 18:00 - 19:00 | NPC晚餐时间 | 部分NPC聚集在广场 |
| 20:00 | 商店关门 | 无法交易 |
| 20:00 - 次日2:00 | 夜晚时段，需要光源或手电筒 | 怪物刷新率增加（地牢外安全） |
| 2:00 | 强制昏倒 | 无论在哪，强制结束一天，结算并跳到次日6:00，扣除部分精力（如果有精力系统） |

**精力系统规则**（基于 `ToolData.energyCost` 字段）：
- 初始最大精力：200点
- 每次使用工具/武器消耗对应能量（例如锄头5点，斧头8点，剑10点）
- 精力归零时无法使用任何工具，只能走路、对话、睡觉
- 恢复方式：
  - 睡觉：恢复全部精力
  - 食物：根据 `FoodData.energyRestore` 恢复
  - 药水：根据 `PotionData.energyRestore` 恢复
  - 泡温泉（如果有）：随时间恢复

### 4.2 主要收入来源与支出

**收入来源**：
1. 出售作物（前期主要）
2. 出售加工品（中期主要）
3. 出售地牢战利品（矿石、钥匙、怪物掉落）
4. 完成任务奖励
5. 钓鱼出售
6. NPC好感送礼回馈（偶尔）

**支出项目**：
1. 购买种子（最大支出）
2. 购买工具升级材料
3. 购买食物/药水（如果自制不足）
4. 解锁配方（一次性费用）
5. 修理工具（如果有耐久系统）
6. 送礼给NPC（提升好感）

### 4.3 前期收支平衡分析（前10天模拟）

### 4.4 经济控制策略

为防止通货膨胀/游戏后期货币泛滥，设计以下经济消耗点：
- **村落复兴捐款**：玩家可捐款加速村落建设（每1000铜币增加1%进度，但后期需要大量）
- **高级配方购买**：隐藏配方需花费5000+铜币
- **工具升级材料**：后期钢锭、金锭需要大量矿石，间接消耗货币（如果买矿石）
- **NPC礼物**：高级礼物需购买，且每个NPC好感度满需要大量送礼
- **地牢门票**：深层地牢需缴纳一定费用（剧情设定为“供奉”）
- **季节限定种子**：高价种子只有特定节日出售，鼓励囤钱

通过以上设计，确保经济系统在30小时游戏时间内不会崩溃。
