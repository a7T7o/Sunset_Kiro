# 篇章四：地牢探索详细策划

## 一、地牢整体架构

### 1.1 地牢设定与命名
**地牢名称**：灰碑深渊 (Grey碑 Abyss)
**入口位置**：村落东北角的废弃矿井（美术使用 Pixel Crawler 的 Tilesets 搭建）
**层数**：6层（对应主线剧情四幕 + 隐藏层）
**每层主题**：

| 层数 | 名称 | 主题 | 怪物来源 | 关键奖励 |
|------|------|------|----------|----------|
| 1 | 遗忘矿道 | 废弃矿洞，骷髅守卫 | Skeleton Crew | 铜矿、记忆碎片·起始 |
| 2 | 兽人兵营 | 兽人占领区，营地结构 | Orc Crew | 铁矿、锁/钥匙系统 |
| 3 | 白骨回廊 | 骷髅军团驻守，迷宫结构 | Skeleton Crew (精英) | 金矿、半自动灌溉图纸 |
| 4 | 记忆回响 | 幻觉与镜像，Boss 镜像 | 特殊（玩家镜像） | 记忆核心碎片 |
| 5 | 时间裂隙 | 时空错乱，裂隙生物 | 混合 + 精英 | 自动化组件 |
| 6 | 锚点核心 | 最终 Boss 领域 | 吞噬者（大型） | 时空锚点 |

### 1.2 地图尺寸与房间类型
- 每层由 5×5 到 7×7 个房间组成（随机布局，但关键房间固定）。
- 房间类型：
  - **普通战斗房**：1-3波怪物，清理后可开宝箱。
  - **精英房**：1个精英怪 + 若干杂兵，掉落钥匙/稀有材料。
  - **宝箱房**：无怪物，但有1-3个宝箱（需钥匙解锁）。
  - **陷阱房**：有机关（箭矢、落石），需躲避（利用现有 SimpleEventData 可实现）。
  - **记忆房**：特殊房间，触发记忆闪回（剧情用）。
  - **Boss房**：每层最深处，击败Boss可进入下一层。

## 三、战斗系统设计

### 3.1 现有框架扩展
目前已有 `PlayerToolHitEmitter` 负责检测玩家攻击命中，但缺少：
- 敌人血量管理
- 敌人AI（移动、攻击、技能）
- 伤害计算
- 死亡掉落

**需新增组件**：
- `EnemyController`：挂载在每个敌人预制体上，管理血量、状态、AI、动画。
- `EnemySpawner`：地牢房间内的怪物生成点。
- `DamageCalculator`：静态类，计算玩家攻击对敌人的伤害（基于武器攻击力、暴击、敌人防御），以及敌人攻击对玩家的伤害（基于玩家装备防御）。

### 3.2 伤害计算公式
**玩家对敌人**：
```
基础伤害 = 武器攻击力 × (1 + 随机浮动 ±10%)
实际伤害 = 基础伤害 - 敌人防御力 (最低为1)
暴击：暴击率由武器 `criticalChance` + 装备/技能加成决定，暴击伤害 = 基础伤害 × (1.5 + 武器 `criticalDamageMultiplier`)
```
**敌人对玩家**：
```
基础伤害 = 敌人攻击力
实际伤害 = 基础伤害 - 玩家防御力 (最低为1)
```
玩家防御力来自装备（`EquipmentData.defense` 字段和 `attributes` 的防御加成）。

### 3.3 战斗操作与反馈
- 玩家使用武器攻击：点击鼠标左键（或按键）触发 `PlayerToolHitEmitter` 的扇形检测，命中敌人后调用 `EnemyController.TakeDamage(damage)`。
- 敌人受击：播放受击动画，血量减少，若血量≤0则播放死亡动画，然后生成掉落物（`WorldItemDrop` 已有）。
- 敌人攻击：当玩家进入敌人攻击范围，敌人播放攻击动画，检测碰撞，命中后玩家掉血（`PlayerController.TakeDamage` 需新增），并触发屏幕震动、伤害数字等UI反馈。
- 玩家死亡：血量归零时触发 `SleepEvent` 类似昏倒，但扣除更多金钱（地牢死亡全掉落）。

### 3.4 地牢内精力消耗
- 每次攻击消耗武器 `ToolData.energyCostPerAttack`（已在 `WeaponData` 中定义）。
- 玩家需注意精力管理，避免在地牢中耗尽精力而无法攻击/逃跑。

### 3.5 逃脱与死亡惩罚
- **逃脱绳**（`ItemData` 不可堆叠）：使用后立即回到地牢入口，消耗绳子。
- **死亡**：如果玩家在地牢中血量归零，触发“全掉落”规则：
  - 背包所有物品掉落在地牢当前房间（生成 `WorldItemDrop`）。
  - 玩家在村落家中醒来，精力为最大值的50%。
  - 可再次进入地牢捡回物品，但若再次死亡或地牢刷新（每天？）则物品消失。
- **凌晨2点昏倒**：同样全掉落（但玩家通常不会拖到2点还在深层地牢）。

## 四、Boss战设计

### 4.1 第4层Boss：记忆镜像 (Memory Mirror)

**素材**：复用玩家角色素材，使用镜像特效（半透明+闪烁）。
**背景**：灰碑的能量投影出的玩家自身，代表玩家内心的恐惧与怀疑。
**战斗机制**：
- 镜像拥有玩家当前装备的复制武器和攻击方式。
- 镜像不会移动，但会瞬移到随机位置。
- 攻击方式：
  - 阶段1：使用玩家主武器攻击（伤害为玩家攻击力的80%）。
  - 阶段2（血量<50%）：召唤2个分身（低血量，攻击力低，但干扰）。
- 特殊机制：镜像每次受到攻击会记录伤害类型，然后下一轮反击时使用相同类型（若玩家用火武器，镜像也会用火）。
- 击败后掉落：记忆碎片·真实（用于解锁最终工作台升级）。

### 4.2 第6层最终Boss：时间裂隙吞噬者 (Time Rift Devourer)

**素材**：可使用 Pixel Crawler 中的大型怪物拼接，或自制（需要新制作但量不大）。
**外观**：由无数钟表齿轮和裂隙组成的巨大生物，半透明。
**战斗机制（三阶段）**：
- **阶段1（100%-70%血量）**：
  - 普通攻击：爪子横扫（范围伤害）。
  - 技能：时间减速 - 玩家移动速度降低50%，持续5秒。
  - 召唤：每30秒召唤2只裂隙猎犬（小怪）。
- **阶段2（70%-30%血量）**：
  - 新增技能：时间裂隙 - 在地面生成多个裂隙区域，玩家站在上面持续掉血。
  - 技能：回溯 - Boss 回到5秒前的位置，并恢复少量生命。
  - 召唤：裂隙猎犬数量增至4只。
- **阶段3（<30%血量）**：
  - 狂暴：攻击速度+50%，攻击力+30%。
  - 技能：时间停止 - 全屏暂停3秒，Boss在此期间可移动攻击（玩家无法操作，只能承受伤害？但可通过特殊道具免疫）。
  - 击败后掉落：时空锚点（剧情物品）。

**击败难度**：需要玩家有全自动化生产的顶级装备（金武器+传说防具），并准备充足的食物和药水。

## 六、钥匙背包博弈详细设计

### 6.1 钥匙类型与获取

| 钥匙类型 | ID范围 | 获取方式 | 开锁成功率加成 | 对应锁类型 |
|----------|--------|----------|----------------|------------|
| 铜钥匙 | 1420-1429 | 普通宝箱、杂货店购买（50金） | +0% | 铜锁（基础60%→最终60%） |
| 铁钥匙 | 1430-1439 | 精英掉落、铁锁箱开出 | +20% | 铁锁（40%→60%），铜锁（60%→80%） |
| 金钥匙 | 1440-1449 | Boss掉落、特殊宝箱 | +40% | 所有锁 |
| 万能钥匙 | 1450-1459 | 传说级任务奖励 | +100% | 必然成功，但数量极少 |

### 6.2 开锁概率公式
```
最终开锁概率 = 锁的基础概率 + 钥匙的加成概率
```
例如：铁锁（40%）+ 铁钥匙（+20%）= 60%成功率。成功则钥匙保留（因为 `KeyLockData.unlockChance` 字段控制的是钥匙消耗概率？已有代码描述：最终概率 = 钥匙开锁概率 + 箱子被开锁概率；成功保留钥匙，失败消耗钥匙。这意味着开锁成功时钥匙不消耗？还是消耗？我们根据描述“成功保留钥匙，失败消耗钥匙”来设计：开锁成功，钥匙保留；开锁失败，钥匙消失。这鼓励玩家使用高成功率钥匙。

**设计澄清**：代码中 `unlockChance` 是钥匙的开锁概率？还是锁的基础概率？字段命名可能指钥匙本身的开锁能力。我们需要统一理解：每个 `KeyLockData` 如果是钥匙，则 `unlockChance` 表示该钥匙的成功率；如果是锁，则 `unlockChance` 表示该锁的基础成功率。最终成功率 = 钥匙成功率 + 锁成功率？这会导致可能超过100%。更合理的做法是：锁有一个基础难度（0-1），钥匙有一个破解能力（0-1），最终成功率 = 钥匙能力 / (钥匙能力 + 锁难度) 之类。但现有代码简单定义为加法，我们只能遵循现有设计：`最终概率 = 钥匙开锁概率 + 箱子被开锁概率`。那么铜钥匙开锁概率假设为40%，铜锁开锁概率为20%，则总60%。如果超过100%则必然成功且钥匙保留。

为了不超100%，我们设定：
- 铜钥匙：`unlockChance=0.4`
- 铁钥匙：`unlockChance=0.6`
- 金钥匙：`unlockChance=0.8`
- 万能钥匙：`unlockChance=1.0`
锁的基础概率：
- 铜锁：`unlockChance=0.2`
- 铁锁：`unlockChance=0.2`（但实际总概率40%）
- 特殊锁：`unlockChance=0.1`（总概率50%配金钥匙）
这样总概率最高1.0（金钥匙+特殊锁=0.9，不会超）。

### 6.3 背包博弈策略

玩家进入地牢前需要决策携带物品，背包格有限（初始12格，可通过背包扩容道具增加，但需占用饰品槽？）。

**典型携带方案对比**：

| 方案 | 携带物品 | 格数 | 风险 | 收益 |

| 保守型 | 3把铜钥匙 + 5食物 + 1逃脱绳 + 3格空 | 12 | 开锁成功率低，可能浪费钥匙 | 安全，能捡少量战利品 |
| 平衡型 | 2把铁钥匙 + 1把金钥匙 + 4食物 + 1逃脱绳 + 4格空 | 12 | 成功率中等 | 可开较高级锁，有格子装战利品 |
| 赌徒型 | 5把金钥匙 + 2食物 + 1逃脱绳 + 4格空（但需先有金钥匙） | 12 | 万一死亡全掉，但开锁必成 | 可开所有锁，格子少但赌大件 |
| 作死型 | 不带逃脱绳，全带钥匙和食物 | 12 | 无法逃脱，死亡全掉 | 最大格子装战利品，但风险极高 |

**死亡全掉落**机制迫使玩家谨慎，不会无脑带满钥匙。

### 6.4 地牢中钥匙的额外用途
- 某些门需要特定钥匙（如铁门需铁钥匙），与宝箱不同，门使用钥匙后永久消失（门解锁后不再需要钥匙）。
- 隐藏房间需要钥匙开启，里面可能有记忆碎片。

## 七、地牢刷新与重置规则

- 每天地牢重置？不，那样玩家可以无限刷。我们采用“敌人每3天刷新，宝箱每月刷新（季节变更）”。
- 但首次通关奖励只能拿一次。
- 钥匙和宝箱是主要资源，需控制产出，防止玩家刷钥匙导致经济崩溃。

**具体规则**：
- 普通怪物：每3天（游戏内）重生。
- 精英怪：每周（7天）重生，掉落钥匙概率降低50%。
- 宝箱：每月（28天）重置，但上锁宝箱需要新钥匙开启。
- Boss：不重生（只能打一次，但可再次进入Boss房探索，无Boss）。

这样玩家不能短期内无限刷钥匙，必须合理安排地牢探索周期。
